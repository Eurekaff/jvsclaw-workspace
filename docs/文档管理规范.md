**版本**: v1.0  
**生效日期**: 2026-04-09

## 📁 文档分类

### 1. 配置文档 (`docs/配置文档/`)

系统配置、架构、规范类文档

**命名规则**: 小写英文，下划线分隔

**包含**:
- `workflow_overview.md` - 工作流概览
- `agent_contracts.md` - Agent 合同规范
- `usage_guide.md` - 使用指南

### 2. 技能文档 (`docs/技能文档/`)

Skills、Tools、集成相关文档

**命名规则**: 小写英文，下划线分隔

**包含**:
- `skills_tools_usage_guide.md` - Skills/Tools 使用指南
- `design_md_integration.md` - Design.md 集成
- `skills_test_report.md` - Skills 测试报告

### 3. 产出文档 (`docs/产出文档/`)

任务执行报告、项目产出文档

**命名规则**: **标准中文命名**

**格式**: `YYYYMMDD_主题_类型.md`

**示例**:
- `20260409_Skills.sh 集成_任务报告.md`
- `20260409_下一步建议_执行报告.md`
- `20260409_Skills-Tools 集成_完成报告.md`

### 4. API 文档 (`docs/API 文档/`)

API 接口、编程指南

**命名规则**: 小写英文，下划线分隔

**包含**:
- 链接到各模块 README

## 📝 命名规范详解

### 配置文档和技能文档

```
<模块名>_<内容类型>.md

示例:
- workflow_overview.md
- agent_contracts.md
- skills_tools_usage_guide.md
```

### 产出文档（标准中文命名）

```
<日期>_<主题>_<类型>.md

日期：YYYYMMDD 格式
主题：使用中文，简洁描述
类型：报告 | 计划 | 总结 | 分析

示例:
- 20260409_Skills.sh 集成_任务报告.md
- 20260409_监控面板_开发报告.md
- 20260409_工作流优化_计划.md
```

## 🔄 文档更新流程

### 新增文档

1. **确定分类**
   - 配置相关 → `配置文档/`
   - 技能相关 → `技能文档/`
   - 任务产出 → `产出文档/`
   - API 相关 → `API 文档/`

2. **按规则命名**
   - 配置/技能：小写英文
   - 产出：标准中文 + 日期

3. **更新索引**
   - 编辑 `docs/INDEX.md`
   - 添加到对应分类

4. **提交 Git**
   ```bash
   git add docs/
   git commit -m "docs: 新增 <文档名>"
   git push
   ```

### 修改文档

1. 直接编辑文档
2. 在文档末尾添加更新记录
3. 提交 Git

### 归档文档

旧文档移动到 `docs/archive/` 目录

## 🎯 最佳实践

### ✅ 推荐

1. **配置文档用英文** - 便于代码引用
2. **产出文档用中文** - 便于人类阅读
3. **日期格式统一** - YYYYMMDD
4. **定期更新索引** - 保持 INDEX.md 最新
5. **文档头部信息** - 包含版本、日期、作者

### ❌ 避免

1. ~~配置和产出混在一起~~
2. ~~命名不统一~~
3. ~~缺少索引~~
4. ~~日期格式混乱~~
